`raiku export` and `raiku import` are complementary commands for capturing and restoring your Raiku package environment using a human-readable plain-text file. `export` writes one line per installed package (name, version, language, and optionally the SHA-256 hash) to `requirements.raiku`. `import` reads that file and installs every package it contains. Together they are the easiest way to share a Raiku environment between teammates, bootstrap CI, or onboard a new developer without requiring them to know which packages to install.

## Usage

```bash
# Export
raiku export [--output FILE] [--language LANG]

# Import
raiku import [--file FILE] [--trust] [--no-build]
```

## Flags

### `raiku export`

| Flag | Short | Default | Description |
|------|-------|---------|-------------|
| `--output` | `-o` | `requirements.raiku` | File to write the package list to. |
| `--language` | `-l` | all | Export only packages for one language. |

### `raiku import`

| Flag | Short | Default | Description |
|------|-------|---------|-------------|
| `--file` | `-f` | `requirements.raiku` | Requirements file to read. |
| `--trust` | | false | Skip build-command confirmation for all packages during import. |
| `--no-build` | | false | Download and cache packages only; skip all build commands. |

## `requirements.raiku` format

The file is plain text â€” one package per line, comments starting with `#` are ignored:

```
# Raiku requirements file
# Generated by: raiku export
# Install with: raiku import

blazing-vec==1.0.0  # Rust  --hash sha256:a3c8f1d2e4b7...
fast-math==1.2.0    # Python  --hash sha256:9f3e2a1c5d8b...
goqueue==0.9.1      # Go
```

Each non-comment line follows the format: `<name>==<version>`. The `# <Language>` comment and `--hash` annotation are written by `raiku export` but are not required for `raiku import` to work â€” the importer looks up version and language from the index at install time.

## Examples

<CodeGroup>
```bash Export all installed packages
raiku export
```

```bash Export to a custom file
raiku export --output my-packages.raiku
```

```bash Export Rust packages only
raiku export --language Rust --output rust-deps.raiku
```

```bash Import from the default file
raiku import
```

```bash Import from a custom file
raiku import --file my-packages.raiku
```

```bash Import without running build commands (CI)
raiku import --no-build
```

```bash Import trusting all packages (automated environment)
raiku import --trust --no-build
```
</CodeGroup>

<Note>
`raiku import` installs packages using their names and versions from the requirements file, going through the standard install flow including hash verification. It does **not** guarantee byte-for-byte identical installs across machines in the way that `raiku from-lock` does. For fully reproducible installs with locked hashes, use [`raiku from-lock`](/cli/from-lock) with a committed `raiku.lock`.
</Note>

<Tip>
Commit `requirements.raiku` to your repository so new contributors can get up and running with a single command:

```bash
raiku import
```

For CI, add `raiku import --no-build` to your setup step.
</Tip>
